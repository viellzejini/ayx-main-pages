# ayx-main-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面作为独立的静态资源存放，不针对任何特定域名或网站。

## 项目简介

`ayx-main-pages` 是一个轻量级的页面归档仓库。主要用途包括：

- 存放和发布独立的 HTML 文件
- 作为静态页面的集中存储点
- 便于版本管理和内容更新

每个 HTML 页面均为独立文件，彼此之间无依赖关系。

## 目录说明

```
├── index.html          # 入口页面（可选）
├── pages/              # 存放主要 HTML 页面的目录
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：放置所有独立 HTML 页面，文件名应具有描述性。
- `assets/`：存放页面所需的公共资源文件，按类型划分子目录。
- 根目录下的 `index.html` 可作为导航或入口页面（可选）。

## 页面归档说明

- 所有 HTML 页面均为静态内容，无需后端服务即可访问。
- 页面之间不应存在硬编码的相互引用，保持独立。
- 如需引用公共资源，请使用相对路径指向 `assets/` 目录。
- 每个页面建议包含必要的 `<meta>` 标签，以明确字符编码与视口设置。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件，并确保其完整性。
- 更新页面：直接修改对应的 HTML 文件，建议在 commit 信息中说明改动内容。
- 删除页面：移除对应文件，并检查是否有其他文件引用该资源。
- 不建议在页面内嵌入过多外部资源链接，以保持归档的稳定性。

## 注意事项

- 本仓库仅供归档和展示用途，不承担页面内容的其他责任。
- 请勿在页面中放置任何形式的自动跳转、恶意代码或违反平台政策的内容。
- 所有提交应保持文件整洁，避免冗余或无关文件。
